Getting Started

System Overview

The INET Operations Command Center manages your telecom rollout projects end-to-end — from the first purchase order to the final invoice — through a single portal. Every stage is tracked here.

Four User Roles

Project Manager (PM)

Full operations view. All teams, all projects, command dashboard, financials, and master data.

Implementation Manager (IM)

Personal dashboard filtered to own teams and projects. Manages materials, execution, and backend team work.

PIC (Invoice Controller)

Tracks MS1 / MS2 milestones, manages invoice pipeline, creates Sales Invoices from the portal.

Field Team

Today's assigned work, on-site execution forms, team stock, and material returns.

ℹWhen you log in, the portal automatically shows the right layout for your role. You only see what is relevant to you.

Project Manager

PO Upload

Brings new purchase orders into the system. Two modes:

Standard Upload (Current POs)

1
Get the PO file
Download the latest export file (.xlsx) from Huawei's procurement system.
2
Upload the file
Drop it onto the upload area or click to browse. The system reads and validates it automatically.
3
Review results
Valid rows are ready to import. Error rows show the reason (unknown item code, missing project, etc.).
4
Confirm Import
Creates work order records grouped by PO number. Status starts as "New".

Archive Upload (Historical Records)

Used to bring in completed or cancelled POs from past periods so dashboards and reports reflect your full business history.

ℹUpload the Master Tracker (.xlsb) file. Only rows already marked Closed or Cancelled are imported — active work is never touched. The import runs in the background; you can safely close the page.

Assigning Backend Work

If your account has backend assignment capability enabled, you will see a Backend page in your navigation. This is for work orders handled by external backend teams (not your own field teams).

1
Find the work order
The Backend page lists all dispatched POIDs eligible for backend assignment.
2
Assign the backend team
Select the backend team from the dropdown and save. Status changes to "Backend Assigned".
3
Mark Work Done
When the partner confirms completion, use the "Mark Work Done" action on the Backend page. The system creates a Work Done record automatically — no Rollout Plan or Execution form is needed for backend work.
ℹThe Backend page is only visible to IMs who have been specifically enabled for backend assignment. If you need this access, contact your Project Manager.

Expense Claim Approvals

Field team leads submit project expense claims that route to you for approval. Open Expense Approvals in your navigation.

1
Pending tab
See all claims from your teams waiting for a decision. Each claim shows the team, date, total amount, and a breakdown by expense type and POID.
2
Review the lines
Expand the claim to see each expense line — type, description, amount, and the work order(s) it was filed against.
3
Approve or Reject
Click Approve to process the claim, or Reject with a reason. The field user is notified of the outcome.
Implementation Manager

Material Management

You manage materials from arrival at the main warehouse through to consumption on site. The full chain: Huawei delivers → IM receipts the delivery → IM requests transfer to team warehouse → field team uses materials on site.

DUID Stock — Main Warehouse Overview

Each delivery from Huawei is identified by a DUID (site delivery code). The DUID Stock tab on the Material Requests page shows:

ColumnMeaning
DUIDSite / delivery code for this shipment
ItemsNumber of item types in this delivery
RemainingItems still available to transfer (received minus already-requested)
Request buttonOnly shown when remaining items exist for this DUID

Creating a Material Transfer Request

1
Find your DUID
Go to Material Requests → DUID Stock tab. Find the DUID with the items you want to transfer.
2
Click Request
A form opens. Select the POID (work order) and destination team. The DUID pre-fills automatically.
3
Review & confirm Huawei Materials (blue section)
Customer-provided items from the Huawei outbound plan auto-populate. These come directly from the Huawei receipt. Adjust quantities downward if you are not transferring everything — you cannot add new items here.
4
Add Company Materials (green section)
INET-owned materials (consumables, tools, hardware not from Huawei). Search by item code or name, select from the dropdown, enter the quantity. The current stock level shows next to each item. This section is optional — leave it empty if only Huawei materials are needed.
5
Submit & Approve
Submit the request. Go to the Requests tab, find the pending request, and click Approve. Stock moves from the main warehouse to the team warehouse.
ℹA single request can contain both Huawei and Company materials at the same time. Both are transferred in one Stock Entry when approved.

Approving Material Returns

When a field team submits a return request from their My Stock page:

1
Open the Returns tab
Material Requests → Returns tab shows all pending return requests from your teams.
2
Review & approve
Check the items and quantities. Click Approve Return to transfer the stock back to the main warehouse.
ℹThe field team's stock page reflects the transfer immediately after approval. No further action is required.

Submitting an Expense Claim

Use the Expense page to submit project-related costs (fuel, accommodation, tools, meals, etc.) for your IM to review.

⚠The + New Claim button only activates when your user account has an Employee record linked. If it appears greyed out, contact your IM.
1
Tap + New Claim
Your team name fills automatically. Set the Claim Date and an optional overall Remarks (e.g. "Jeddah North site visit, 3 days").
2
Add expense lines
For each cost tap + Add Expense Line. Each line needs: Expense Type (Travel, Accommodation, Meals, etc.), Description, Amount (SAR), and a POID — the POID field is required for every line.
3
Single POID or Multi-POID
Each line links to one work order by default. Toggle to Multi-POID to split the cost across several work orders. The system shows a split preview: SAR [amount] ÷ [N] POIDs = SAR [amount each] — the total is divided equally.
4
Submit Claim
The running total updates as you type. Tap Submit Claim to send it to your IM for review.

The claims list has four tabs:

Each card shows the claim date, number of lines, POID count, and total SAR. Colour coding: blue = pending, amber = approved/unpaid, green = paid, red = rejected. Tap a card to see the full detail including the IM’s rejection reason if applicable.

Field Team

My Stock & Material Returns

View all materials currently in your team’s warehouse. Stock is broken down by DUID (the delivery code for each Huawei shipment) so you always know the origin of each item.

Reading Your Stock

Each item shows as a card. The left border colour tells you the material type:

BadgeMeaning
HuaweiCustomer-provided material from a Huawei outbound delivery
Company (INET)INET-owned material (consumables, tools, hardware not supplied by Huawei)

Expand a card to see the DUID breakdown — which Huawei delivery codes the current stock came from and the quantity per delivery.

Returning Materials

If you have unused, excess, or incorrectly delivered materials to return to the main warehouse:

1
Tap Return Materials
All items currently in your stock are listed with checkboxes.
2
Check items & confirm quantities
Check the items you want to return. Each item pre-fills the full available quantity — reduce it if returning only part. The system prevents you from entering more than you have.
3
Enter a reason & submit
Required — describe why (unused, damaged, wrong delivery, project cancelled, etc.).
4
IM approves
Your IM reviews the request. Once approved, stock moves back to the main warehouse automatically. Track the request status in the Returns History below the stock list.

PIC — Invoice Controller

PIC Tracker

Your main day-to-day working page. Every active work order line is here. Filter, sort, edit, and bulk-update statuses.

What you can do

Filter & Sort

Filter by project, IM, site, billing status, or date range. Sort any column. Totals row at the bottom sums all monetary columns over your current view.

Edit a line

Click any row to open the edit panel. Milestones are separated into MS1, MS2, and Acceptance tabs so editing is focused. Fields are validated before saving — for example, dates cannot be set in the future.

Bulk Update

Select multiple lines and change their status in one action — e.g. mark 20 lines as "Submitted to Operator (MS1)" at once.

Download CSV

Exports your current filtered and sorted view.

Common Questions

Dashboard shows old data
Press Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac) to force a full page refresh
I can't see my teams / projects
Contact your system administrator to verify your user account is set up correctly
PO Upload row shows an error
Verify the item code and project code exist in the masters; contact your PM if unsure
Work Done billing status stuck on Pending
The PIC team updates this as they process milestones — no action needed on your side
Material Request button not showing for a DUID
All received items for that DUID have already been requested. Remaining qty is zero.
IM dashboard empty or showing wrong data
Contact your system administrator to link your user account to your IM record and teams
Invoice Tracker shows "Ready" lines but no invoice created
Select the lines and click Create Sales Invoice. Check that the customer and item details are complete if it fails.
Return request submitted but stock not moved
The IM needs to approve the return request. Stock only moves after IM approval.
IM does not see a Backend page
The Backend page is only shown to IMs with backend assignment enabled. Contact the Project Manager to enable it on your IM record.
Expense claim submission fails with employee error
Your Frappe user account must be linked to an Employee record. Ask your system administrator to link it.
Expense claim does not appear in IM Approvals list
The claim routes to the IM assigned to your team. Verify your team has a correct IM assigned.
