The field portal is designed for mobile use on site. Pages use a card-based layout optimised for touch and small screens.

4.1 Today's Work /pms/today

Your home screen. Shows all work your team has planned for today.

Header: Time-based greeting ("Good morning/afternoon/evening, [your first name]"), today's full date (e.g. "Monday, 26 May 2026"). Refresh button.

Summary chips: Total (plan count) · In Progress (amber dot) · Planned (blue dot) · Done (green dot).

Plans are sorted by urgency: In Progress first → Planned → Completed.

Plan card colours (left border)

Border colourStatus
AmberIn Progress / In Execution
BluePlanned (not started)
GreenCompleted
GrayCancelled

Card contents

  • Item description (large title), POID (bold) + Plan name (small, muted)
  • IM ✓ badge (green, top right) — shown if your IM has confirmed this execution
  • Site code + site name, Activity type, Project code, Visit type + your share qty, Access time + period
  • QC Not Required / CIAG Not Required amber badges (when IM unchecked those toggles)
  • IM note callout — if your IM left a Manager Remark, it appears as a highlighted callout at the bottom of the card

Card footer: "Tap to execute →" (if not started) or "Continue execution →" (if In Progress). Tap any card to open the Execution Form.

4.2 Execute /pms/field-execute

The main on-site execution form. Opens from Today's Work or directly via URL. Without a plan ID, shows a list of all actionable plans for your team.

Timer section

  • Start Timer (play icon) — starts the time log for this plan. Only one timer can run at a time across all your plans.
  • Stop Timer (stop icon) — stops the running timer and logs the time.
  • If a timer is running on a different plan, a warning shows: "A timer is running on [other plan name]". Stop it here before starting this one.

Plan information header

Shows: Item description, POID, site code, activity type, project, visit type, target qty. IM Note callout — Manager Remark shown as an amber/blue callout above the form fields. Plan Teams Breakdown — all teams on this plan and their assigned quantities.

Form fields

FieldNotes
Execution Status (required)In Progress · Completed · Hold · Cancelled · Postponed
Achieved QuantityPre-filled from plan target. Adjust if actual quantity differs.
QC StatusOnly shown if QC is required. Options: Pending / Pass / Fail / N/A
CIAG StatusOnly shown if CIAG is required. Options: Open / Submitted / Accepted / Rejected
GPS LocationClick Capture GPS to read device location ("Lat, Lon (±Xm)"). Or type coordinates manually.

Team Lead Remark (TL Remark)

  • Remark picker (searchable dropdown): click to open saved remark templates. Type to filter. Selected remarks appear as chips; click × to remove. If your text doesn't match any template, tap "+ Add '[text]' as new remark" to save it as a template and pick it simultaneously. Multiple remarks can be combined.
  • Extra free-text textarea — for anything not covered by templates; appended below picked remarks on save.

Previously saved remarks from this plan pre-fill on load so re-editing doesn't lose prior text.

Materials section

Shows all materials transferred to your team warehouse for this POID. For each item: Item name, item code, Status badge ("In Warehouse" green or "Pending" amber), Transferred qty, Used qty input (editable, max = transferred qty, pre-filled with transferred qty — adjust downward if you used less).

Inline Expense section

Log site expenses directly from the execution form without opening the Expense page separately. For each expense line: Expense Type (required), Amount SAR (required, >0), Description (optional), + Add button. Queued lines shown in a table below. Submitted together with the execution form.

Photos / Attachments

  • Camera button — opens the device camera or file picker
  • Uploaded photos show as thumbnail previews immediately
  • Each photo has a Remove button (×). Upload progress shows pending thumbnail greyed out.

Submit

The Save button submits all fields: execution status, achieved qty, QC/CIAG status, GPS, TL remarks, material usage, inline expense lines, and photos.

When status is Completed, the system automatically creates a Work Done record. After a successful submission: "All Done!" message. Navigate back to Today's Work using the back arrow.

4.3 QC / CIAG /pms/field-qc-ciag

Quality check and Customer Inspection and Acceptance (CIAG) forms — for cases where QC/CIAG needs to be completed independently from the execution form.

  • QC Status select (Pass / Fail / N/A) + QC Remark textarea + photo upload
  • CIAG Status select (Submitted / Accepted / Rejected / Open) + CIAG Remark textarea + photo upload
  • Save & Complete button per section

4.4 History /pms/field-history

Read-only log of all previous work completed by your team. Toolbar: Date range, Activity Type filter, Status filter. Export Excel. Columns: Plan name, POID, Item Description, Site, Project, Visit Type, Plan Date, Status badge, Achieved Qty, Remarks.

4.5 My Stock /pms/field-my-stock

View all materials currently in your team's warehouse and submit material return requests. Stock is updated automatically when your IM approves a transfer — no refresh or confirmation needed.

Stock item card layout

  • Item Name (large, bold) + Item code (monospace, small)
  • Quantity (large, right-aligned) + UOM (e.g. Nos, Meters)
  • Material type badge: Huawei (customer-provided from Huawei delivery) · Company (INET) (INET-owned material)
  • Left border colour: amber = Huawei · blue = Company (INET)
  • Tap "▼ X DUIDs" to expand and see which delivery codes this stock came from

Returning Materials

1
Tap Return Materials
Button at the top of the stock page. A form opens listing all items currently in your stock.
2
Check items to return
Checking an item pre-fills the full available quantity — reduce it if returning only part. The system will not allow a return quantity greater than your current stock.
3
Enter a Reason (required)
Describe why: unused, damaged, wrong delivery, project cancelled, etc.
4
Tap Submit
Return request sent to your IM for approval. After IM approval, stock moves back to the main warehouse automatically.

Return request statuses

StatusMeaning
PendingSubmitted, awaiting IM approval
ApprovedIM approved, stock transferred back
RejectedIM rejected, stock stays in your warehouse

4.6 Expenses /pms/field-expense

Submit expense claims for project-related costs (fuel, accommodation, meals, tools, etc.).

⚠️
The + New Claim button is only active if your user account has an Employee record linked. If greyed out, contact your IM.

Your claims — four tabs

TabShows
PendingSubmitted claims waiting for IM decision (blue border)
UnpaidApproved but payment not yet made to you (amber border)
PaidFully settled claims (green border)
AllComplete claim history

Creating a New Claim

1
Tap + New Claim
Opens the claim form.
2
Fill the header
Date (defaults to today, required) · Team (pre-filled, read-only) · Remarks (optional overall note).
3
Add expense lines
Tap + Add Expense Line for each cost. Required: Expense Type (Travel, Accommodation, Meals, etc.), Amount SAR (>0), POID. Optional: Description.
4
Single vs Multi-POID mode
Each line defaults to Single POID. Toggle to Multi-POID to link the cost to multiple work orders. Live split preview: SAR [amount] ÷ [N] POIDs = SAR [amount each]. Total divided equally across all selected POIDs.
5
Tap Submit Claim
Claim sent to your IM. You cannot edit it after submission. Running total shown as you add lines.

Claim status flow

Submitted→ Pending (IM review)→ Approved→ Unpaid→ Paid

Or: Pending → Rejected (rejection reason visible to you)
