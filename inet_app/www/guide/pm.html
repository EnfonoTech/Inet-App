+ Create Project modal: Project Code (required), Status, Project Name (required), Customer, Implementation Manager, Center/Area, Project Domain, Huawei IM, Budget Amount (SAR).

2.4 PO Upload /pms/po-upload

Two modes: Standard Upload (current POs) and Archive Upload (historical records).

Standard Upload — three-step wizard

1
Upload
Select Customer (required), drop the .xlsx file onto the upload area. The system validates automatically and moves to Review.
2
Review
Summary chips: Valid rows (green), Error rows (red), Total rows (amber), New products (warning), Missing projects (error). Click Confirm Import (disabled if any error rows exist).
3
Success
Green banner showing Lines Imported, Duplicates Skipped, New POs, Appended POs. Upload Record table with search and status filter.

Archive Upload: Upload the Master Tracker .xlsb file. Preview shows CLOSED (green), CANCELLED (red), OPEN (gray, skipped). Click Start archive import — runs in background. Job status panel shows progress.

💡
Run Standard Upload first for the current month, then Archive imports for prior months in chronological order.

2.5 PO Dump /pms/po-dump

Full export of all PO line data. Filters: From/To date, Status checkboxes (OPEN/CLOSED/CANCELLED — update instantly), text search. Status cells colour-coded: OPEN=blue, CLOSED=gray, CANCELLED=red. Download CSV button.

2.6 Dispatch /pms/dispatch

Three tabs: Pending Dispatch (New lines), Dispatched, All Lines.

Toolbar: Text search (POID, PO No, Item, Project, DUID), All Projects, All IMs, All DUIDs, Date range, Clear, Dispatch Selected (N) button.

Dispatched / All tabs show additional columns: Mode badge (Auto=violet, Manual=slate), IM, Target Month. Auto-dispatched rows have a subtle violet background.

Dispatch Modal

  • Assign IM dropdown (required)
  • Planning Mode dropdown — Plan / Direct (required)
  • Summary info box: selected line count → IM name → Mode

Convert to Manual button (when Auto-dispatched lines are selected): converts selected Auto lines to Manual with optional Re-assign IM.

2.7 Planning /pms/planning

Scope toggle: "Unplanned" (default) · "All POIDs (re-plan)" for adding additional visits.

Create Plans modal

  • Lead team dropdown (required)
  • Visit type: Execution / Re-Visit / Extra Visit
  • Additional Teams (optional) — "+ Add team" to split work with Qty input. Summary shows Total qty · Assigned to extras · Remaining for lead (red if over-allocated).
  • Access Details: Planned start date (required), Planned end date (required, ≥ start), Access time, Access period (Not set / Day / Night)
  • Workflow toggles: QC Required (checked by default), CIAG Required (checked by default)
  • Remark textarea — visible to field team and IM
Visit TypeRevenue MultiplierWhen to use
Execution (Work Done)1.0×Standard first visit
Re-Visit0.5×Return to site for rework
Extra Visit1.5×Additional mobilisation required

2.8 Execution Monitor /pms/execution

Read-only live monitor of all execution activity across all teams. 27-column table including: Plan, POID, Dummy POID, Item Code, Description, Activity Type, Project, DUID, Center area, Region, Team, IM, Plan Date, Visit Type, Visit #, Target, Achieved (colour-coded: green ≥80%, amber ≥40%, red <40%), Plan Status, TL Status (clickable badge), Exec Status, QC, CIAG, Issue Category (clickable button), General/Manager/Team Lead Remarks (inline edit), Open (View button).

2.9 Work Done /pms/work-done

Summary cards: Executed Qty (total) · Revenue (SAR total).

Table columns: POID, Item Code, Description, Activity Type, Project, DUID, Team, Executed Qty, Revenue, Billing Status badge, General Remark (inline edit), Submission Status (clickable), Open (View button).

Billing StatusMeaning
PendingWork complete, billing not yet started or still in progress
InvoicedCommercial invoice submitted to customer
ClosedPayment received, milestone fully closed

Billing status is set automatically by PIC actions — no manual editing needed here.

2.10 Issues & Risks /pms/issues-risks

Track and manage execution issues across all teams. Full toolbar with Issue Category, Exec Status, TL Status, QC Status, CIAG Status, Projects, Teams, DUIDs, Date range filters. Selection + "Create Plans (N)" button — Visit Type pre-set to "Re-Visit".

2.11 Backend /pms/backend

Manages work assigned to backend teams. Two tabs: Pending · Work Done. Selection + Mark Work Done button — enter Completion Date (required) + Remark. Creates a synthesised Work Done record for each selected line.

2.12 Reports /pms/reports

TabContent

2.13 Time Logs /pms/timesheets

Summary cards: Log lines (blue) · Total hours (green). Toolbar: Text search (user, plan, team, project), Team ID filter, Date range, Clear. Export Excel. Columns: ID, User, Team, Rollout (Plan link), Start Time, End Time, Duration (hours), Notes.

2.14 Team Allocation Approvals /pms/approvals

Review and approve or reject Team Allocation Requests from IMs. A red dot badge on the Approvals nav link shows pending count. Two tabs: Awaiting Approval · History. Decision modal shows full request details with optional Remark textarea, Approve and Reject buttons.

2.15 Expenses /pms/expenses

View all expense claims submitted by field teams across all IMs and teams. Same interface as IM Expense Approvals (§3.12) but unscoped — shows everything. Total SAR calculated across filtered rows shown in the toolbar.

2.16 Operations Overview /pms/overview

TabWhat it shows

2.17 Material Requests /pms/im-material-request

PM view of all material transfer requests across all IMs. Identical to the IM Material Management page (§3.10) but unscoped — useful for monitoring that transfers are happening on schedule.

2.18 Masters /pms/masters

Reference data management. Card grid with 12+ master types, each with: icon, description, live record count badge, expandable preview table, + New button (opens Frappe Desk form).

Master types: Area, INET Team, IM Master, Project Domain, Huawei IM, Subcontractor, Customer Item, Activity Cost, Item Catalog, Project, Customer, Visit Multiplier, Subcontractor Rates, Execution Category.

Deep link: /pms/masters?expand=<DocType Name> opens a specific card already expanded.
