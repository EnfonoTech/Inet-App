Section 01

System Overview

The INET Operations Command Center replaces the Excel-based project management system. It manages the full lifecycle of telecom field operations through a 6-stage pipeline:

PO Upload
→
Dispatch
→
Planning
→
Execution
→
Work Done
→
Dashboard KPIs

Three User Roles

Admin

Full Command Dashboard on wall screen. All pipeline stages, reports, master data.

Implementation Manager

Personal dashboard filtered to own teams and projects.

Field Team

Today's assigned work. Submit execution updates with GPS.

ℹThe system auto-detects your role at login. Admins = System Manager role. IMs = Full Name matches INET Team IM field. Everyone else = Field Team.

Section 03

PO Upload

URL: /pms/po-upload — Import Huawei PO export files.

1
Get the PO file
Download .xlsx from Huawei's procurement system
2
Drag & drop the file
Drop onto upload zone or click to browse. System uploads and parses automatically.
3
Review validation
Valid rows (Item Code found, Project exists, Qty > 0). Error rows with details.
4
Confirm Import
Creates PO Intake records grouped by PO number. Status starts as "New".
Section 04

PO Dispatch

URL: /pms/dispatch — Assign PO lines to field teams.

1
Select PO lines
Table shows imported lines (status "New"). Use checkboxes to select.
2
Choose Team & Target Month
Select team from dropdown (Team-01 to Team-60). Pick target month.
3
Dispatch Selected
Creates PO Dispatch records with unique System ID. Auto-maps IM and Customer.
⚠The System ID is the primary tracking key — it follows the work through planning, execution, and completion.
Section 05

Rollout Planning

URL: /pms/planning — Schedule dispatched work.

1
Select dispatched lines
Shows PO Dispatch records with status "Dispatched".
2
Set plan date & visit type
Work Done (1.0x), Re-Visit (0.5x), or Extra Visit (1.5x).
3
Create Plans
Target Amount = PO Line Amount × Visit Multiplier. Status becomes "Planned".
Visit TypeMultiplierWhen Used
Work Done1.0xStandard first visit
Re-Visit0.5xReturn visit for rework
Extra Visit1.5xAdditional mobilization
Section 06

Execution Monitor

URL: /pms/execution — Real-time view of field progress (read-only for admin). Auto-refreshes every 30 seconds. Shows System ID, Team, Plan Date, Target, Achieved, Completion %, Status.

Section 07

Work Done

URL: /pms/work-done — Completed, billable work with auto-calculated financials.

ColumnSource
Revenue (SAR)Billing Rate × Qty from Customer Item Master
Cost (SAR)Team Daily Cost + Subcontract Cost
Margin (SAR)Revenue − Cost
ℹRate logic: Standard Region → Standard_Rate. Hard Region → Hard_Rate.

Section 13

First-Time Setup

Import Master Data

1
Upload CONTROL_CENTER.xlsx
Frappe Desk → File Manager → Upload
2
Run import script
cd ~/frappe-bench
bench --site mysite.local console

from inet_app.api.data_import import import_control_center_xlsx
result = import_control_center_xlsx("/files/CONTROL_CENTER.xlsx")
print(result)
frappe.db.commit()
3
Verify at /pms/masters
7 areas, 60 teams, 7 subcontractors, 74 projects, 4 multipliers

Set Up Users

IMs: Create users with Full Name matching INET Team im field (e.g., "Majid Ali"). They auto-get the IM Dashboard.

Field Teams: Create users for team leads. They default to Field role.

Section 14

Troubleshooting

Dashboard shows old white theme
Hard refresh: Cmd+Shift+R or incognito window
"Loading..." stuck forever
Check server: cd ~/frappe-bench && bench serve --port 8001
PO Upload validation errors
Verify Item Codes and Project Codes match master data
Dashboard KPIs all zero
Complete the full pipeline: upload → dispatch → plan → execute
Login fails
Use Frappe credentials. Default: Administrator / admin
IM Dashboard wrong data
User's Full Name must match im field in INET Team exactly
