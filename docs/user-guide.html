Section 01

System Overview

The INET Operations Command Center replaces the Excel-based project management system. It manages the full lifecycle of telecom field operations through a 6-stage pipeline:

PO Upload
→
Dispatch
→
Planning
→
Execution
→
Work Done
→
Dashboard KPIs

Three User Roles

◆

Admin

Full Command Dashboard on wall screen. All pipeline stages, reports, master data.

★

Implementation Manager

Personal dashboard filtered to own teams and projects. Performance tracking.

⚙

Field Team

Today's assigned work. Submit execution updates with GPS and photos.

ℹ The system auto-detects your role at login based on your Frappe user account. Admins = System Manager role. IMs = Full Name matches INET Team IM field. Everyone else = Field Team.

Section 03

PO Upload

URL: /pms/po-upload — Import Huawei PO export files into the system.

Get the PO file
Download the .xlsx export from Huawei's procurement system (e.g., PURCHASE_ORDER_20260325003415.xlsx)
Drag & drop the file
Drop onto the upload zone or click to browse. System uploads to Frappe and parses the file.
Review validation results
Green rows = valid (Item Code found, Project Code exists, Qty > 0). Red rows = errors with details shown.
Click "Confirm Import"
Creates PO Intake records grouped by PO number. Each PO becomes one record with line items. Status starts as "New".
ℹ Columns parsed: PO No, PO Line No, Shipment No, Site Name, Site Code, Item Code, Item Description, Unit, Qty, Unit Price, Line Amount, Project Code, Project Name, Center Area, Publish Date
Section 04

PO Dispatch

URL: /pms/dispatch — Assign imported PO lines to field teams.

Select PO lines
Table shows imported lines with status "New". Use checkboxes to select lines for dispatch.
Choose assignment
Team: Select from dropdown (Team-01 through Team-60). Target Month: When this work should be done.
Click "Dispatch Selected"
Creates PO Dispatch records with unique System ID (e.g., SYS-2026-00001). Auto-maps IM from Team Master and Customer from Project Master.
⚠ The System ID is the primary tracking key — it follows the work through planning, execution, and completion.
Section 05

Rollout Planning

URL: /pms/planning — Schedule dispatched work for specific dates.

Select dispatched lines
Table shows PO Dispatch records with status "Dispatched".
Set plan date & visit type
Plan Date: When the team should execute. Visit Type: Work Done (1.0x), Re-Visit (0.5x), or Extra Visit (1.5x).
Click "Create Plans"
Creates Rollout Plan records. Target Amount = PO Line Amount × Visit Multiplier. Status becomes "Planned".

Visit Multipliers

TypeMultiplierWhen Used
Work Done1.0xStandard first visit
Re-Visit0.5xReturn visit for rework
Extra Visit1.5xAdditional mobilization needed
Plan0xPlanning only, no cost

Execution Monitor

Section 13

First-Time Setup

Import Master Data

Upload CONTROL_CENTER.xlsx to Frappe
Go to Frappe Desk → File Manager → Upload the file
Run the import script
cd ~/frappe-bench
bench --site mysite.local console

from inet_app.api.data_import import import_control_center_xlsx
result = import_control_center_xlsx("/files/CONTROL_CENTER.xlsx")
print(result)
frappe.db.commit()
Verify at /pms/masters
Check record counts match: 7 areas, 60 teams, 7 subcontractors, 74 projects, 4 multipliers

Set Up IM Users

Create Frappe users for each IM. Set their Full Name to match the im field in INET Team master (e.g., "Majid Ali", "Ajmal Muhammed"). They'll automatically get the IM Dashboard on login.

Set Up Field Team Users

Create Frappe users for team leads. They default to "Field" role and see Today's Work filtered to planned items.

Section 14

Troubleshooting

Dashboard shows old white theme
Hard refresh: Cmd+Shift+R or open in incognito window
"Loading..." stuck forever
Check bench server: cd ~/frappe-bench && bench serve --port 8001
PO Upload validation errors
Verify Item Codes match Customer Item Master and Project Codes match Project Control Center
Dashboard KPIs all zero
No Work Done records yet — complete the full pipeline: upload → dispatch → plan → execute → complete
Login fails
Use Frappe credentials. Default admin: Administrator / admin
IM Dashboard shows wrong data
User's Full Name must match the im field in INET Team master exactly
